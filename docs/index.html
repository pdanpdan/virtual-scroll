Virtual Scroll

API Reference

Complete documentation for @pdanpdan/virtual-scroll.

Introduction

@pdanpdan/virtual-scroll is a high-performance Vue 3 virtual scroll library designed to handle massive lists with ease. It supports vertical, horizontal, and bidirectional (grid) scrolling, dynamic item sizes using ResizeObserver, and integration with the browser's native window scroll.

Usage Modes

Compiled Component

The recommended way for most projects. Uses pre-compiled JS for faster builds.

import { VirtualScroll } from '@pdanpdan/virtual-scroll';
import '@pdanpdan/virtual-scroll/style.css';
  • Compatible with all modern bundlers.
  • Note: Manual CSS import is required.
  • Scoped CSS is preserved via data-attributes.

Original Vue SFC

Import the raw source for custom compilation and optimization.

import VS from '@pdanpdan/virtual-scroll/VirtualScroll.vue';
  • Enables better tree-shaking in your build.
  • Styles are handled automatically by your Vue loader.
  • Better source-map debugging in some IDEs.

CSS Classes

The component provides several CSS classes for easy styling and overrides:

  • .virtual-scroll-container: The root container element.
  • .virtual-scroll-wrapper: The element wrapping all items.
  • .virtual-scroll-item: Individual item elements.
  • .virtual-scroll-header / .virtual-scroll-footer: Header and footer elements.
  • .virtual-scroll-loading: Loading slot container.
  • .virtual-scroll--sticky: Applied to header, footer, or items when they are in a sticky state.
  • .virtual-scroll--hydrated: Applied to the container once it has mounted and initialized on the client.
  • .virtual-scroll--window: Applied when scrolling via the browser window/body.
  • .virtual-scroll--table: Applied when containerTag="table" is used.
  • .virtual-scroll--debug: Applied when debug mode is active.

Events

scroll

Emitted when the container is scrolled.

Payload (ScrollDetails<T>):

  • currentIndex: Index of the first visible row.
  • currentColIndex: Index of the first visible column.
  • scrollOffset: { x, y } current scroll position.
  • viewportSize: { width, height } size of the container.
  • totalSize: { width, height } total scrollable size.
  • isScrolling: Boolean indicating if scrolling is in progress.
  • isProgrammaticScroll: True if scroll was triggered by scrollToIndex or scrollToOffset.
  • range: { start, end } indices of currently rendered rows.
  • columnRange: { start, end, padStart, padEnd } indices and padding for rendered columns.
  • items: Array of currently rendered items with their offsets and sizes.

load

Emitted when scrolling near the end of the content.

Payload (direction):

Either 'vertical' or 'horizontal' depending on which dimension triggered the load.

visibleRangeChange

Emitted whenever the rendered items range or column range changes (due to scrolling or resize).

Payload (range):

{ start: number, end: number, colStart: number, colEnd: number }

Keyboard Navigation

When the component container has focus, it supports the following navigation shortcuts:

  • Home / End: Scroll to the beginning or end of the list.
  • ArrowUp / ArrowDown: Scroll up or down by 40px.
  • ArrowLeft / ArrowRight: Scroll left or right by 40px.
  • PageUp / PageDown: Scroll up or down (or left/right depending on direction) by one viewport height/width.

Exposed Methods

scrollToIndex(row, col, options?)

Scrolls the container to bring a specific item into view.

  • row: Row index (or null to ignore).
  • col: Column index (or null to ignore).
  • options: Can be an alignment string, a ScrollAlignmentOptions object, or a ScrollToIndexOptions object.
    • If it's a string ('start' | 'center' | 'end' | 'auto'), it sets the alignment for both X and Y axes. Default behavior is 'smooth'.
    • If it's a ScrollAlignmentOptions object ({ x?: ScrollAlignment, y?: ScrollAlignment }), it sets independent alignment for each axis.
    • If it's a ScrollToIndexOptions object:
      • align (default: 'auto'): Can be a string or a ScrollAlignmentOptions object.
      • behavior (default: 'smooth'): 'auto' or 'smooth'.

scrollToOffset(x, y, options?)

Scrolls to a specific pixel offset. Pass null to keep current position on an axis.

  • x: Horizontal offset in pixels.
  • y: Vertical offset in pixels.
  • options: { behavior?: 'auto' | 'smooth' } (default behavior is 'auto').

refresh()

Resets all dynamic measurements and re-initializes sizes from props. Useful if the items' content changed significantly.

useVirtualScroll Composable

The core logic for custom implementations. It handles all calculations and state management for virtualization. 

import { useVirtualScroll } from '@pdanpdan/virtual-scroll'
// props must be a Vue Ref
const { renderedItems, scrollToIndex } = useVirtualScroll(props)

Functions

scrollToIndex(row, col, options?)

Programmatically scrolls to a specific row and/or column.

  • row: number | null | undefined - Row index.
  • col: number | null | undefined - Column index.
  • options: ScrollAlignment | ScrollAlignmentOptions | ScrollToIndexOptions. Default behavior is 'smooth'.

Returns: void

scrollToOffset(x, y, options?)

Programmatically scrolls to a specific pixel offset.

  • x: number | null | undefined - Horizontal offset.
  • y: number | null | undefined - Vertical offset.
  • options: { behavior?: 'auto' | 'smooth' }. Defaults to { behavior: 'auto' }.

Returns: void

updateItemSize(index, width, height, element?)

Updates the stored size of an item. Should be called when an item is measured (e.g., via ResizeObserver).

  • index: number - The item index.
  • width: number - Measured width in pixels.
  • height: number - Measured height in pixels.
  • element: HTMLElement (optional) - The element that was measured. Used in grid mode to auto-detect column widths from children.

Returns: void

updateHostOffset()

Recalculates the position of the host element relative to the scroll container. Useful if the container or host moves without a scroll event.

Returns: void

getColumnWidth(index)

Helper to get the width of a specific column based on the current columnWidth prop configuration.

  • index: number - The column index.

Returns: number - The width in pixels.

refresh()

Resets all dynamic measurements and re-initializes from props.

Returns: void
